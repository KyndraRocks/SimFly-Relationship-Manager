Help & Guide
What this app does

SimFly Relationship Manager Lite tracks the give-and-take of flights between you and other SimFly players. It shows who's flown to your airports recently, who you've returned visits to, and who's overdue for a reciprocation — so you can prioritise whom to fly to next. Everything is stored locally on your machine: the app never sends data anywhere.

🚀 Getting started — three things to set up
  1. Pick a save file. Click 📁 Choose save file… in the top bar (Chrome/Edge only). The app will autosave a JSON file there every time you make a change — your authoritative copy on disk. In other browsers, use ⬇ Export to download a snapshot whenever you want a backup.
  2. Point at your Active Airports app. Click ✈ Set AA file… in the top bar and paste the path or URL to your SimFly_Active_Airports-LATEST.html file (the Launch / Send to AA buttons need to know where it is). Get Active Airports at github.com/KyndraRocks/SimFlyActiveAirports.
  3. Bring in your flight history. Either use Quick Entry to type pilots in one at a time, or use the ↓ Sync Inbound Flights from SimFly panel for a bulk import — see the next section.
↓ Sync Inbound Flights from SimFly

The fastest way to get up to date. Two-step DevTools workflow:

  1. Step 1 — Console snippet. While logged in to simfly.io, open DevTools (F12 → Console tab). Click ⎘ Copy snippet on the Sync panel, paste into the console, press Enter. A textarea appears on the SimFly page with the JSON output.
  2. Step 2 — Paste JSON. Copy the JSON, paste into the Sync panel's Step 2 textarea (or click ⎙ Paste to pull from clipboard), then click ↓ Import.

New flights are added to the log, duplicates are skipped, and any pilots not yet in your roster are auto-created (with a 3-letter code derived from the username). Each imported flight gets the destination airport ICAO appended to its row, and the flight that first introduced a pilot wears a green NEW pill.

✈ Quick Entry

For one-off entries.

  1. Type a pilot's name or 3-letter code in the Pilot field. A dropdown appears — use arrow keys or click to select.
  2. Press Enter (or click Log Flight ↵) to record an inbound flight and reset for the next.
  3. The Session log below shows everything you've added this session, each with an Undo button.
  4. Shortcut: in SimFly, click Copy Link on a pilot's name and paste the whole URL (e.g. https://simfly.io/logbook/<name>/<id>) directly into the Pilot field. The username and SimFly profile URL are extracted automatically.
🆕 Adding a new pilot

If a pilot isn't in your roster yet, the Quick Entry dropdown offers + Add "[name]" as new pilot. The mini-form lets you set the 3-letter code (auto-suggested), optionally the first airport ICAO, and a SimFly profile URL.

↩ Logging a return flight

When you fly to a pilot's airport, click ↩ Recip in their row. This stamps a return flight against that pilot and resets the reciprocation date — so the Balance Queue reflects activity since your last visit.

⚖ Balance Queue (the main view)

A divergent bar chart showing every tracked pilot's inbound visits to you (right, cyan) vs your outbound returns to them (left, green). All pilots are shown — no cap.

  • Click a row to expand — see airport ICAOs and the ✈ DEP, ✈ ARR, ↩ Recip, and ✎ Edit buttons. Click again to collapse.
  • Sort headers — click Pilot, ← Outbound, or Inbound → to sort. Click again to reverse direction.
  • Search — narrow by pilot name, 3-letter code, or airport ICAO.
  • Date range — 3d / 7d / 14d / 30d / 90d / 1yr / All preset buttons or custom from/to dates.
  • Hide inactive — hides pilots with zero activity in the selected window.
  • Has airports — hides pilots with no registered airports (can't be reciprocated to).
  • ☐ Select for AA — enter selection mode, pick the pilots you want to focus on, then 🗺 Send to AA → opens Active Airports' region map for just those pilots.
🗺 Plan Flight → Active Airports

Click ✈ Dep or ✈ Arr on any pilot row to set them as departure or arrival. A Route banner appears at the top showing the route and a ✈ Open in Active Airports button. Clicking it opens AA in a new tab pre-loaded with your route. The first time you click, you'll be asked where your AA HTML file is — paste the path or URL once and it's saved for future launches.

📊 Recip Dashboard

Click 📊 Dashboard in the toolbar for a detailed history view with date-range presets at the top.

  • Summary chips — outbound returns, pilots reciprocated, distinct destinations, inbound visits, pilots visiting you, last visit timestamps.
  • ↩ Top Pilots Visited — horizontal bar chart, top 20 by count.
  • 📍 Top Destinations — top 20 destination ICAOs with their owning pilot.
  • ↩ Reciprocation List — every return flight, filterable by pilot, sortable, exportable as CSV.
  • ⤓ Backfill from SimFly — collapsible at the bottom for seeding historical reciprocations using a console snippet against /api/user/flights. Safe to re-run.
📁 Pilot Directory

Click Pilot Directory in the toolbar to manage SimFly profile URLs for everyone — both tracked pilots and the broader owner list (anyone who has airports but you haven't logged a flight from yet). The Track → button promotes an owner to a tracked pilot.

🔗 Pilot profile links

Pilot names in the main table and analytics are clickable links to that pilot's SimFly profile.

  • Cyan dotted underline — a profile URL is set. Click to open in a new tab.
  • Amber dashed underline — no URL yet. Click to paste one inline.
  • Set or change the URL anytime from ✎ Edit on a pilot's row.
💾 Saving your data

Your data is stored in three places: live in memory, in your browser's localStorage (always written synchronously), and — if you've connected one — a local disk file via the File System Access API.

  • 📁 Choose save file… — pick a JSON file. The app autosaves to it every change (debounced ~2s). Chrome/Edge only. Permission may need re-grant on reload — click ▶ Resume if the bar asks.
  • ⬇ Export — download a dated JSON snapshot. Works in every browser. Use this for periodic backups.
  • ⬆ Import — load a JSON file. Choose Merge (union by id/timestamp, nothing is lost) or Replace All (discard current data, file becomes the source).
  • ✈ Set AA file… — tell the app where your Active Airports HTML file lives.
⌨ Keyboard shortcuts
  • Enter — select pilot from dropdown / submit flight entry
  • ↑ ↓ — navigate pilot dropdown
  • Escape — close dropdown or dismiss modal

⤓ Backfill from SimFly (run console snippet → paste JSON)
Step 1 — Run on simfly.io

While logged in to simfly.io, open the browser DevTools (F12), switch to the Console tab, paste this snippet, and press Enter. It auto-detects your username, walks every page of /api/user/flights with a small jitter delay, and copies a JSON dump to your clipboard. Progress logs every page; expect ~30 seconds per 100 flights.

(async()=>{const s=await fetch('/api/user/session',{credentials:'include'}).then(r=>r.json()).catch(()=>null);const u=s&&s.user_login;if(!u){console.error('Not logged in to simfly.io.');return;}console.log('SimFly reciprocation extractor — user:',u);const out=[];let p=1,tp=1;do{const url=p===1?`/api/user/flights?username=${encodeURIComponent(u)}`:`/api/user/flights?fpage=${p}&username=${encodeURIComponent(u)}`;const d=await fetch(url,{credentials:'include'}).then(r=>r.json()).catch(e=>{console.error('Page '+p+' failed',e);return null;});if(!d)break;tp=d.totalPages||1;for(const f of(d.flights||[])){if(!f.mission_start_ts||!f.destination_icao)continue;out.push({ts:f.mission_start_ts,departureIcao:f.departure_icao||'',destinationIcao:f.destination_icao});}console.log(`Page ${p}/${tp} — ${(d.flights||[]).length} flights · running total ${out.length}`);p++;if(p<=tp)await new Promise(r=>setTimeout(r,250));}while(p<=tp);const j=JSON.stringify(out);console.log(`Done. ${out.length} flights extracted (${(j.length/1024).toFixed(1)} KB JSON).`);try{await navigator.clipboard.writeText(j);console.log('✓ JSON copied to clipboard. Paste into Recip Tracker → 📊 Dashboard → Backfill from SimFly → Step 2.');}catch(e){window.simflyFlightSeed=j;console.warn('Clipboard blocked. Run: copy(window.simflyFlightSeed)');}return{extracted:out.length};})();
Step 2 — Paste & Import

Paste the JSON the snippet copied. Each entry should look like {"ts":"2026-03-28T04:04:21+00:00","departureIcao":"PAFA","destinationIcao":"PACD"}. The importer matches each destinationIcao against the airports owned by pilots in your roster — flights to airports owned by tracked pilots become reciprocations; flights to other airports (training, sightseeing, missions to non-roster airports) are skipped. Duplicate entries (same ts + pilot) are not added twice. The legacy {ts, pilotUsername, airports} shape is also accepted.

v1.0.3 May 29, 2026 · Current Fix
  • Footer airport count now matches the database. The "N owners · N airports" footer stat was summing raw ICAO arrays from the Gist, including duplicates across owners and ICAOs not present in the airport database. The count now deduplicates by ICAO and excludes entries that don't resolve, matching the set of airports the app can actually look up.
v1.0.2 May 17, 2026 Improved
  • Balance Queue expand arrow now points down when expanded — the per-row expand indicator on the Balance Queue showed ▶ when collapsed and ▲ when expanded. The expanded state now uses ▼ instead, matching the standard convention that a downward triangle indicates an open/expanded disclosure (and matching the chevron behavior already used in the Quick Entry log).
v1.0.1 May 15, 2026 Fix
  • Hotfix: AA inbox no longer shared with the original Recip Tracker — v1.0.0 inherited the RT inbox-drainer code that reads localStorage['simfly_recip_inbox'], which Active Airports' "Add to Recip Tracker" flow writes to. With both RT and RM Lite installed in the same browser, whichever tab loaded first would drain the queue and the other would miss the pilot. Lite now uses its own simfly_rmlite_inbox queue, so the two apps stay fully independent. (AA doesn't write to the Lite key today, so Lite's inbox is empty in vanilla setups — the drainer is hot-wired for future cross-app handoff but never steals from RT.)
  • CSV export filename branded — the Reciprocation List CSV download now uses simfly_rmlite_returns_… instead of the inherited simfly_recip_returns_….
v1.0.0 May 15, 2026 Initial
  • Initial release of SimFly Relationship Manager Lite — a redistributable, gist-free fork of the original SimFly Recip Tracker. Tracks the give-and-take of flights between you and other SimFly players, surfaces who's overdue for a return visit, and launches Active Airports pre-loaded with your route or selected owners. Every byte of data lives locally on your machine.
  • Local-first storage — data is always written to localStorage on every change as a safety net. Choose a JSON file via the 📁 Choose save file button (Chrome/Edge File System Access API) and the app autosaves to it on every change. Other browsers fall back to manual ⬇ Export / ⬆ Import. No GitHub PAT, no cloud sync, no external storage.
  • Import flow — load any prior export with the choice of Merge (union by ID and timestamp; nothing is lost) or Replace All (discard current data, file becomes the source). Confirms before either path.
  • Balance Queue — the main view: a divergent bar chart of inbound visits vs outbound returns per pilot, with date-range presets, sorting, search, and a Select for AA mode that sends the chosen pilots' airports to Active Airports' region map.
  • Quick Entry · Sync Inbound Flights · Cutoff — type one pilot at a time, or bulk-import every inbound flight since your last sync via a DevTools console snippet on simfly.io.
  • Pilot Directory — manage SimFly profile URLs for every tracked pilot and every airport owner; promote owners to tracked pilots with a single click.
  • Recip Dashboard — date-filtered history: top pilots visited, top destinations, full reciprocation list with CSV export, and a backfill-from-SimFly snippet for seeding historical data.
  • Active Airports launch — ✈ Dep / ✈ Arr on any row sets a route; the route banner's launch button opens Active Airports pre-loaded. First-launch prompt asks where your local AA file lives; the path is saved and the ✈ Set AA file button on the save bar lets you change it anytime.
  • First-run welcome modal — three-step intro shown on initial load (save file, AA path, bring in flights). Dismissable; one-time only.
  • Help modal — full guide accessible from the toolbar ? button.
